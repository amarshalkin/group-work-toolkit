# group-work-toolkit

Плагин Claude Code для полного цикла работы с групповой деятельностью в неформальном образовании: **проектирование → ревью → проведение → разбор**.

Методологическая база — Конструкторы сообществ практики (КСП). Предусмотрено расширение под другие методологии (дизайн-мышление, Liberating Structures) и другие роли (модератор / организатор / заказчик).

**Версия:** v0.10.0 · апрель 2026
**Автор:** Артём Маршалкин
**Базируется на:** скилле `group-work-design` v0.9, прогнанном на трёх кейсах в iteration-1.

---

## Что плагин делает

Покрывает три фазы работы над сессией:

1. **Проектирование.** Скилл `design-group-work` → на выходе Инструкция модератору и Карта ожидаемых результатов.
2. **Ревью перед проведением.** Скилл `review-session-design` → прогон готового проекта по чек-листу, отчёт «светофор».
3. **Разбор после проведения.** Скилл `debrief-session` → сравнение плана с фактом, выводы для следующей итерации плагина.

Каждый скилл триггерится как по естественному запросу пользователя, так и по явной slash-команде.

## Состав

### Скиллы

| Скилл | Что делает | Slash-команда |
|---|---|---|
| `design-group-work` | Проектирует групповую работу с нуля — от 2-часового такта до многодневного события | `/group-work-toolkit:design-session` |
| `review-session-design` | Ревью готового проекта по чек-листу, отчёт в формате «светофор» | `/group-work-toolkit:review-design` |
| `debrief-session` | Пост-сессионный разбор: план vs факт, улучшения для iteration-2 | `/group-work-toolkit:debrief` |

### Команды

Три slash-обёртки в `commands/` — лёгкие точки входа для явного вызова скиллов.

### Evals

5 тест-кейсов в `evals/evals.json`:
- 3 кейса для `design-group-work` (перенесены из v0.9 без изменений).
- 1 кейс для `review-session-design` — неполный черновик Инструкции, должен быть пойман 🔴 или 🟡 с конкретными фиксами.
- 1 кейс для `debrief-session` — пост-сессионный разбор педсовета-2026 с улучшениями для плагина.

## Установка

### Локальный запуск для тестирования

Из корня `педсовет-весна-2026/`:

```bash
claude --plugin-dir group-work-toolkit
```

Или через `cc`:

```bash
cc --plugin-dir ./group-work-toolkit
```

### Глобальная установка

Скопируй папку `group-work-toolkit/` в `~/.claude/plugins/` и перезапусти Claude Code.

## Использование

### Проектирование

**Автотриггер:** «спроектируй сессию на 3 часа для 15 учителей», «сделай постановку модератору», «разработай программу форсайта».

**Явный вызов:**

```
/group-work-toolkit:design-session педсовет, 20 учителей, 3 часа, тема «ИИ в школе»
```

Скилл задаст два обязательных вопроса — масштаб и глубина методологии — и соберёт вводные через интерактивные вопросы.

### Ревью проекта

**Автотриггер:** «проверь эту инструкцию модератору», «готова ли сессия», «пройди по чек-листу».

**Явный вызов:**

```
/group-work-toolkit:review-design path/to/moderator-brief.md
```

### Разбор после сессии

**Автотриггер:** «разбор после сессии», «что получилось на педсовете», «сравни с картой результатов».

**Явный вызов:**

```
/group-work-toolkit:debrief path/to/results-map.md + заметки модератора
```

## Расширение

Плагин спроектирован так, чтобы его можно было наращивать, не ломая работающее ядро.

### Добавить методологию

Положи файл в `skills/design-group-work/references/methodologies/` по шаблону из `methodologies/README.md`. Добавь evals-кейс. Не меняй базовые референсы КСП.

### Добавить роль

Положи файл в `skills/design-group-work/references/roles/` по шаблону из `roles/README.md`. При необходимости добавь новый шаблон артефакта в `skills/design-group-work/assets/`.

### Добавить скилл

Создай папку `skills/<skill-name>/` с `SKILL.md`. В описании укажи специфические триггеры. Если нужен slash-вызов — добавь обёртку в `commands/`.
