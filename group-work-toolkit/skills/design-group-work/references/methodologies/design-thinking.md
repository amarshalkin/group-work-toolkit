# Design Thinking — заглушка

**Статус:** не заполнено в v0.10. Не использовать.

Этот файл — место под контент в следующих итерациях плагина `group-work-toolkit`. Пока пользователь просит спроектировать сессию в логике дизайн-мышления (empathize → define → ideate → prototype → test), перенаправь в одну из существующих базовых схем КСП:

- «Проблемы → Задачи → Решения» — ближайший аналог (см. `../base-schemas.md`).
- Либо работай в режиме «Свободный стиль» и опиши процесс простыми словами без методологических отсылок к дизайн-мышлению.
