# Чек-лист проверки проекта сессии

Пройди по этому списку до того, как отдавать Инструкцию модератору. Если хоть один пункт красный — возвращайся и дорабатывай.

## Рамка и цели

- [ ] Объект сформулирован (что меняется в мире по итогам сессии).
- [ ] Предмет сформулирован (о чём конкретно говорим на сессии).
- [ ] Рамка сформулирована (масштаб, горизонт, ограничения).
- [ ] Объект, предмет, рамка не путаются между собой.
- [ ] Сформулирован ожидаемый результат сессии — конкретно, с формой упаковки.
- [ ] Понятно, кому и зачем передаётся результат после сессии.

## Логика сценария

- [ ] Выбрана базовая схема (практика-теория-практика / вызов-обсуждение-игропрактика / проблемы-задачи-решения / форсайт / матрёшка).
- [ ] Каждый такт вносит вклад в общий результат — не висит в воздухе.
- [ ] Сумма тактов даёт ожидаемый результат сессии.
- [ ] Между тактами видны переходы: что приходит из предыдущего и что уходит в следующий.
- [ ] В последнем такте есть сборка.
- [ ] В сессии есть рефлексия (отдельный такт, не внутри сборки).

## Такт за тактом

Для каждого такта проверь:

- [ ] Сформулирован конкретный измеримый результат такта (что группа произведёт к концу).
- [ ] Указана форма коллективной работы (обсуждение, игропрактика, доклад...).
- [ ] Указан размер группы (не более 20 на обсуждение, не более 7 на проектную работу).
- [ ] Указана норма высказывания.
- [ ] Указана очерёдность (по руке, по очереди, по жребию).
- [ ] Заложено время на все 5 шагов модерации (постановка, технология, работа, презентация, рефлексия).
- [ ] Рефлексия занимает минимум 10% такта.
- [ ] Длительность такта 1,5-3 часа. Если меньше — это упражнение, не такт.

## Участники

- [ ] Определены категории участников (роли, позиции, статусы).
- [ ] Число участников на каждой форме работы адекватно (7-20 на обсуждение и т.п.).
- [ ] Учтено, что часть участников может быть пассивной — есть механизм включения.
- [ ] Учтено, что часть участников может быть "трудной" — есть страховочные ходы.

## Модератор

- [ ] Понятно, какого типа модератор нужен (формалист / эксперт / коуч / глобалист / эмоционал / сторителлер).
- [ ] Модератор готов к сессии: знает тему, имеет заготовки, прогонял ключевые моменты.
- [ ] У модератора есть печатная инструкция.
- [ ] Прописаны страховочные ходы: зависла группа, конфликт, сбой тайминга, доминатор, бунт.

## Тайминг

- [ ] В дне не более 6-7 часов чистой работы.
- [ ] Есть кофе-брейк каждые 2-3 часа.
- [ ] Обед 60-90 минут, если сессия больше 5 часов.
- [ ] Заложен буфер 10-15% на непредвиденное.
- [ ] Тяжёлая содержательная работа не падает на послеобеденную "яму" (14-16:00).
- [ ] Презентации групп не дольше 3 минут каждая.
- [ ] Доклады не длиннее 20 минут без активности в середине.

## Форма и материалы

- [ ] Для каждого такта указаны материалы (флипчарт, стикеры, карточки, шаблоны).
- [ ] Есть образец итогового продукта или описание, как он должен выглядеть.
- [ ] Для каждого такта понятно, где и как фиксируется результат.
- [ ] Сборка произойдёт в формате, который запросил заказчик.

## Пространство

- [ ] Размер зала адекватен количеству участников (25 м² на группу из 10).
- [ ] Для каждой малой группы предусмотрено место (отдельная комната или зона).
- [ ] Есть зона кофе-брейка вне рабочей зоны.
- [ ] Проверены техника, акустика, вентиляция.
- [ ] Есть резерв на случай отказа техники.

## Для заказчика

- [ ] Заказчик согласовал объект, предмет, рамку.
- [ ] Заказчик знает, какой формат получит на выходе.
- [ ] Заказчик понимает, что он/она делает во время сессии (роль).
- [ ] Заказчик понимает, как результат переводится в действие после сессии.

## Проверка на перегруз

- [ ] Не более 3 задач в одном такте.
- [ ] Не более 5 тактов в дне.
- [ ] Не чаще 1 раза в час смена формата "большая группа ↔ малая".
- [ ] Между тактами — минимум 10 минут буфера на переход.

## Проверка на недогруз (обратная сторона)

- [ ] В каждом такте группа реально что-то производит, а не "обсуждает вообще".
- [ ] На каждый такт нужен именно модератор, а не просто ведущий (если нет — возможно, это не групповая работа, а лекция).
- [ ] Участники в каждом такте достаточно вовлечены (высказываются, работают с материалом).

## Стоп-сигналы

Если любой из этих пунктов — остановись и пересмотри сценарий:

- Нельзя сформулировать ожидаемый результат в одном предложении.
- Модератор сам не может объяснить, зачем нужен такой-то такт.
- Группы по 25+ человек на обсуждение без дробления.
- Нет рефлексии вообще.
- Заказчик не знает, что будет с результатом.
- Сценарий занимает всё время без буфера.
