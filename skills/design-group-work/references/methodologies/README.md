# Методологии проектирования групповой работы

Этот подкаталог содержит индексные файлы по методологиям, на которые может опираться `design-group-work`. Подкаталог — точка расширения: сюда кладутся новые методологии, не трогая базовую логику `SKILL.md`.

## Текущий состав

- `ksp.md` — **Конструкторы сообществ практики**. Работоспособна. Является методологической базой плагина v0.10. Содержит сводку и ссылки на ключевые референсы верхнего уровня (`base-schemas.md`, `tact-structure.md`, `moderator-roles.md`, `reflection-and-assembly.md`).
- `design-thinking.md` — **заглушка**. Не использовать в v0.10. Нужна как место под контент в следующих итерациях.
- `liberating-structures.md` — **заглушка**. Не использовать в v0.10.

## Как добавить новую методологию

1. Создай файл `<methodology-name>.md` по шаблону ниже.
2. Убедись, что методология совместима с базовым аппаратом КСП (объект/предмет/рамка, такт, сборка) — либо явно опиши, в чём расходится и как это отражается на артефактах.
3. Добавь методологию в список в `SKILL.md` как четвёртый вариант ответа на Вопрос 2 («Насколько опираемся на методологию КСП?»), либо как отдельное измерение выбора.
4. Добавь хотя бы один evals-кейс в `../../evals/evals.json` (один из уже существующих кейсов → переформатировать под новую методологию).
5. Прогони iteration на новом кейсе.

### Шаблон файла методологии

```markdown
# <Название методологии>

## Суть
Одно-два предложения: что это, для каких задач.

## Ключевые понятия
- Понятие 1 — короткое определение.
- Понятие 2 — короткое определение.

## Как соотносится с КСП
Где пересекается (такты, объект/предмет/рамка),
где расходится (другие форматы, другой язык).

## Базовые схемы / протоколы
Список протоколов/схем работы, которые даёт методология.
Если есть карты-шпаргалки — отдельный файл рядом.

## Когда использовать
Сигналы, по которым эта методология подходит под запрос пользователя.

## Влияние на артефакты
Как меняется Инструкция модератору и Карта результатов,
если проектируем в этой методологии.

## Источники
Книги, сайты, авторитетные материалы.
```

## Правила совместимости

- Методология **не должна** ломать базовые референсы КСП — они остаются в корне `references/` как общий аппарат.
- Если методология требует своего словаря (например, Liberating Structures — "invitations", "microstructures"), этот словарь остаётся внутри файла методологии и не протекает в `SKILL.md`.
- Переключатель глубины методологии в `SKILL.md` (Глубоко КСП / КСП как опция / Свободный стиль) — общий для всех методологий. Не создавай параллельных переключателей.
