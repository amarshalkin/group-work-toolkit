# Конструкторы сообществ практики (КСП)

## Суть

Методология проектирования и проведения групповой работы для неформального образования, выросшая из школы системомыследеятельностного подхода и адаптированная под образовательные сообщества. Фокус — на **осмысленном результате группы** и на роли модератора как держателя процесса, содержания и перевода в действие.

## Ключевые понятия

- **Объект** — то, что меняется в реальном мире за пределами сессии. См. `../object-subject-frame.md`.
- **Предмет** — то, о чём группа ведёт разговор.
- **Рамка** — контекст, в котором рассматривается предмет (горизонт, масштаб, ограничения).
- **Такт** — законченный блок работы с постановкой, технологией, работой, презентацией и рефлексией. См. `../tact-structure.md`.
- **Норма высказывания** — договорённость о том, как группа высказывается (короткое, по делу, уважительно, один говорит — другие слушают и т. п.).
- **Ранжирование идей** — механизм выбора из списка через голосование, весовые коэффициенты, кластеризацию.
- **Сборка** — финальный шаг такта/блока/дня/события, где произведённое собирается в единый результат. См. `../reflection-and-assembly.md`.
- **Роли модератора** — шесть ролей (Капитан, Переводчик, Катализатор, Сталкер, Иной, Просветитель) и триада Коричина (Предприниматель, Организатор, Технолог). См. `../moderator-roles.md`.

## Базовые схемы (в этом плагине)

Пять рабочих схем проектирования лежат в `../base-schemas.md`:

1. Практика → Теория → Практика.
2. Вызов → Обсуждение → Игропрактика → Обсуждение.
3. Проблемы → Задачи → Решения.
4. Форсайт-схема (тренды → угрозы → карта проектов).
5. Матрёшка для дня (пленар → малые группы → такты внутри группы → сборка в пленар).

## Базовые нормативы

- **Малая группа** — 7–20 человек для обсуждения, 3–7 для проектной работы. См. `../timing-and-group-dynamics.md`.
- **Такт** — 1,5–3 часа, всегда завершается рефлексией (≥10% времени такта).
- **Сборка дня** — обязательна, иначе участники уносят хаос.

## Когда использовать

Когда запрос пользователя — про образовательное событие с группами, содержательную работу над общим объектом (сообществом, практикой, стратегией), когда нужна не просто коммуникация, а **производство результата** группой.

КСП не подходит для массовых лекций без групповой части, для 1-на-1 коучинга, для чисто информационных событий.

## Влияние на артефакты

- **Инструкция модератору** — структура по тактам, речевые заготовки в методологическом языке, ссылки на роли и нормы. См. шаблон `../../assets/moderator-brief-template.md`.
- **Карта ожидаемых результатов** — на каждый такт отдельный блок, явное «что забирают в понедельник». См. `../../assets/results-map-template.md`.
- **Страховочные ходы** — опираются на «страшилки» (`../scary-stories.md`) и работу с трудными участниками (`../difficult-participants.md`).

## Переключатель глубины

В `SKILL.md` (Шаг 1) пользователь выбирает, насколько строго применять КСП:

- **Глубоко КСП** — полный словарь, жёсткая структура.
- **КСП как опция** — методологический скелет с упрощённым языком.
- **Свободный стиль** — методологический аппарат внутри скилла, но не в артефактах; язык простой.

Этот переключатель определяет, какой словарь пойдёт в Инструкцию модератору.

## Источники

- Руководство модератора групповой работы (внутренний документ Центра Знание, педсовет-весна-2026).
- Школа Щедровицкого (СМД-подход) — историческая основа.
- Работы А. Коричина по триаде ролей.
